# Интеграция формы в B2B-проект

При переносе логики формы из этого проекта (facade_factory) в B2B нужно повторить всё то же самое, но с **`source: 'b2b'`**.

## Что взять из facade_factory

1. **Форма (React)**  
   - Управляемые поля, валидация (имя ≥2 буквы, телефон только цифры 10–12, комментарий до 500 символов).  
   - При отправке: `POST` на свой API с телом, где обязательно **`source: 'b2b'`**.  
   - Блок успеха поверх формы на 5 секунд.

2. **API отправки в Telegram**  
   - Роут типа `POST /api/send-to-telegram`: принимает `name`, `phone`, `message`, `role` (если есть), **`source`**.  
   - Использует `BOT_TOKEN` и `CHAT_ID` из `.env`.  
   - В сообщении в Telegram выводит источник (B2B/B2C) и данные в моноширинном виде (`<code>`) — копирование по нажатию на каждый блок в Telegram.

3. **.env в B2B**  
   - Те же переменные: `BOT_TOKEN`, `CHAT_ID`.

## Главное отличие от B2C

- В теле запроса с формы всегда передавать **`source: 'b2b'`** (здесь в B2C передаётся `source: 'b2c'`).  
- Остальная логика (валидация, успех, API) — такая же, как в facade_factory.

## Файлы-образцы в facade_factory

- `src/components/sections/Form/Form.tsx` — форма + валидация + вызов API с `source: 'b2c'`.  
- `src/app/api/send-to-telegram/route.ts` — отправка в Telegram, поддержка `source`.

Когда откроешь B2B-проект, можно сказать: «Внедри форму как в facade_factory по инструкции B2B_FORM_INTEGRATION.md, с source: b2b».
